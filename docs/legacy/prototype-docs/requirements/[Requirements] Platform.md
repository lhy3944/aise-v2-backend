## FR-PF-02: 프로젝트 관리

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-PF-02-01 | 시스템은 프로젝트를 생성할 수 있는 인터페이스를 제공해야 한다. | |
| FR-PF-02-02 | 시스템은 프로젝트 생성 시 이름, 설명, 도메인, 제품 유형을 입력할 수 있어야 한다. | |
| FR-PF-02-03 | 시스템은 프로젝트 생성 시 사전 정의된 5가지 프리셋(All / Requirements Only / Requirements+Design / Requirements+Testcase / Testcase Only) 중 하나를 선택할 수 있어야 한다. | |
| FR-PF-02-04 | 시스템은 선택된 모듈에 해당하는 메뉴만 프로젝트 내에서 표시해야 한다. | |
| FR-PF-02-05 | 시스템은 프로젝트 설정에서 모듈 프리셋을 변경할 수 있어야 한다. | |
| FR-PF-02-06 | 시스템은 프로젝트 정보를 수정할 수 있는 인터페이스를 제공해야 한다. | |
| FR-PF-02-07 | 시스템은 프로젝트를 삭제할 수 있는 기능을 제공해야 한다. (삭제 시 확인 절차 포함) | |
| FR-PF-02-08 | 시스템은 사용자가 참여한 모든 프로젝트를 목록으로 조회할 수 있어야 한다. | |
| FR-PF-02-09 | 시스템은 프로젝트 목록에서 프로젝트 상태, 최근 수정일, 멤버 수 등의 요약 정보를 표시해야 한다. | |

## FR-PF-03: 멤버 관리

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-PF-03-01 | 시스템은 프로젝트에 멤버를 초대할 수 있는 기능을 제공해야 한다. (이메일 초대) | |
| FR-PF-03-02 | 시스템은 멤버에게 역할(Owner, Editor, Viewer)을 부여할 수 있어야 한다. ※ 향후 Reviewer 역할 확장 가능 | |
| FR-PF-03-03 | 시스템은 역할별 권한을 다음과 같이 구분해야 한다: Owner(전체 관리 + 멤버 관리), Editor(요구사항 작성/수정 + SRS 생성 + Review + Export), Viewer(읽기전용) | |
| FR-PF-03-04 | 시스템은 멤버의 역할을 변경할 수 있어야 한다. | |
| FR-PF-03-05 | 시스템은 프로젝트에서 멤버를 제거할 수 있어야 한다. | |
| FR-PF-03-06 | 시스템은 프로젝트 멤버 목록을 조회할 수 있어야 한다. | |

## FR-PF-05: Import / Export (공통 모듈)

> SRS System, TC System(독립모드) 등에서 공통으로 사용하는 Import/Export 기능.
> 기존 FR-RQ-05에서 플랫폼 공통 모듈로 이동.

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-PF-05-01 | 시스템은 PDF, Markdown, Word, Excel, PPT 형식의 문서를 업로드할 수 있는 인터페이스를 제공해야 한다. | |
| FR-PF-05-02 | 시스템은 업로드된 문서에서 텍스트, 테이블, 구조(섹션/목차) 정보를 파싱하여 구조화된 텍스트로 변환해야 한다. | |
| FR-PF-05-03 | 시스템은 파싱된 문서 내용을 사용자가 미리보기(Preview)할 수 있는 인터페이스를 제공해야 한다. | |
| FR-PF-05-04 | 시스템은 Jira에 있는 PRD/SRS 데이터를 가져올 수(Retrieve) 있어야 한다. | |
| FR-PF-05-05 | 시스템은 Polarion에 있는 PRD/SRS 데이터를 가져올 수(Retrieve) 있어야 한다. | |
| FR-PF-05-06 | 시스템은 Import 실패 시 실패 원인을 사용자에게 명확히 표시해야 한다. | |
| FR-PF-05-07 | 시스템은 작성된 산출물들을 PDF, Markdown 형식으로 내보낼 수 있어야 한다. | |
| FR-PF-05-08 | 시스템은 작성된 산출물들을 Word 형식으로 내보낼 수 있어야 한다. | |
| FR-PF-05-09 | 시스템은 작성된 산출물들을 Jira로 업로드할 수 있어야 한다. | |
| FR-PF-05-10 | 시스템은 작성된 산출물들을 Polarion으로 업로드할 수 있어야 한다. | |

## FR-PF-06: 설정

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-PF-06-01 | 시스템은 프로젝트별 LLM 모델을 선택할 수 있는 설정을 제공해야 한다. | |
| FR-PF-06-02 | 시스템은 생성 산출물의 기본 언어(한국어, 영어 등)를 설정할 수 있어야 한다. | |

| FR-PF-06-05 | 시스템은 사용자별 Confluence PAT를 저장/관리할 수 있어야 한다. | |
| FR-PF-06-06 | 시스템은 Polarion 연동 시 AISE 시스템 계정의 PAT를 프로젝트 단위로 설정할 수 있어야 한다. | |
| FR-PF-06-07 | 시스템은 다이어그램 생성 도구(PlantUML, Mermaid 등)를 선택할 수 있는 설정을 제공해야 한다. | |
| FR-PF-06-08 | 시스템은 알림 설정(Review 요청, 멤버 초대 등)을 관리할 수 있어야 한다. | |
