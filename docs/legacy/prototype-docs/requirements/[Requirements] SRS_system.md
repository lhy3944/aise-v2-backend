## FR-RQ-01: 요구사항(FR/QA/Constraints) 및 Assist

> 요구사항 입력은 두 가지 모드를 지원한다:
> - **구조화 모드** (+버튼): 사용자가 항목을 직접 입력하면 AI가 프롬프트 증강으로 후보를 제시, 사용자가 선택
> - **대화 모드** (Chat): 자연스러운 대화를 통해 요구사항을 탐색적으로 정의. AI는 자연스러운 대화체로 응답하며, 사용자 요청 시 대화 내용에서 FR/QA/Constraints를 추출하여 타입별 체크박스로 제안
> - 두 모드의 결과는 동일한 요구사항 목록에 반영되며, 사용자는 모드를 자유롭게 전환할 수 있다.
>
> **섹션(그룹핑)**: 요구사항은 타입별(FR/QA/Constraints)로 섹션(1단계)으로 묶을 수 있다.
> - 섹션은 타입 내에서만 유효하며(예: FR 섹션에는 FR만 소속), 드래그로 순서 변경 및 섹션 간 이동이 가능하다.
> - 섹션이 없으면 기존처럼 플랫 리스트로 동작하며, 미분류 요구사항도 허용된다.
> - 섹션 구조는 SRS 문서 생성 시 챕터 제목으로, Import/Classification 시 자동 추출에 활용된다.

## FR-RQ-02: Review 기능

> 요구사항 리뷰는 Include(포함) 상태인 요구사항을 대상으로 수행한다.
> - 사용자가 "리뷰" 버튼을 클릭하면, `is_selected=true`인 요구사항만 리뷰 대상에 포함된다.
> - Include된 요구사항이 없으면 리뷰를 실행할 수 없다.
> - 리뷰 버튼은 모드(구조화/대화) 전환과 무관하게 상단에 고정 배치된다.
>
> **v1**: 충돌(conflict) + 중복(duplicate) 검출. 각 이슈에 대해 간단한 해결 힌트(1줄)를 제공한다.
> 이슈가 있어도 다음 단계 진행은 가능하되, 경고를 표시한다.
> 마지막 리뷰 결과 1건이 DB에 저장된다.
> **추후 확장**: 모호성(ambiguity) 검출은 사용자 피드백 후 추가 예정.

## FR-RQ-06: Classification

> Import된 문서를 FR/QA/Constraints/Other로 분류하는 기능. 사용자가 문서 Import 후 별도로 Classification을 요청한다.
> 대용량 문서(약 10페이지 이상)는 2-pass 방식으로 처리한다: 1차에서 문서 구조를 파악하고, 2차에서 섹션별로 요구사항을 추출한다.

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-RQ-06-01 | 시스템은 Import된 문서들에 대해 사용자가 Classification을 별도로 요청할 수 있는 인터페이스를 제공해야 한다. |  |
| FR-RQ-06-02 | 시스템은 Import된 문서 내용을 분석하여 FR로 분류하는 기능을 제공해야 한다. |  |
| FR-RQ-06-03 | 시스템은 Import된 문서 내용을 분석하여 QA로 분류하는 기능을 제공해야 한다. |  |
| FR-RQ-06-04 | 시스템은 Import된 문서 내용을 분석하여 Constraints로 분류하는 기능을 제공해야 한다. |  |
| FR-RQ-06-05 | 시스템은 FR/QA/Constraints에 해당하지 않는 내용(배경, 가정, 인터페이스 등)을 Other(기타)로 분류해야 한다. |  |
| FR-RQ-06-06 | 시스템은 Other로 분류된 항목을 사용자가 다른 분류로 재지정할 수 있는 인터페이스를 제공해야 한다. |  |
| FR-RQ-06-07 | 시스템은 Classification 수행 시, 먼저 문서의 목차/섹션 구조를 파악하는 1차 분석을 수행해야 한다. |  |
| FR-RQ-06-08 | 시스템은 1차 분석에서 파악된 섹션 단위로 요구사항을 추출하고 분류하는 2차 분석을 수행해야 한다. |  |
| FR-RQ-06-09 | 시스템은 섹션별 추출 결과에서 중복된 요구사항을 탐지하고, 사용자에게 병합을 제안할 수 있어야 한다. |  |
| FR-RQ-06-10 | 시스템은 Classification 결과에 대해 신뢰도(confidence)를 표시하고, 낮은 신뢰도 항목을 별도로 표시해야 한다. |  |
| FR-RQ-06-11 | 시스템은 Classification 결과를 요구사항 편집 화면(FR/QA/Constraints/Other)에 자동 반영하고, 사용자가 항목별로 승인/수정/삭제할 수 있어야 한다. |  |
| FR-RQ-06-14 | 시스템은 Classification 수행 시 문서의 섹션/제목 구조를 분석하여 요구사항 섹션(그룹)을 자동 생성하고, 추출된 요구사항을 해당 섹션에 배치해야 한다. (FR-RQ-01-30 참조) |  |
| FR-RQ-06-12 | 시스템은 대용량 문서 처리 시 현재 진행 단계(1차 구조 분석 / 2차 섹션별 추출)와 진행률을 실시간으로 표시해야 한다. |  |
| FR-RQ-06-13 | 시스템은 문서로부터 분류를 실패한 경우 사용자에게 명확한 이유를 제시해줘야 한다. |  |

## FR-RQ-07: SRS 문서 생성

> 확정된 요구사항(FR/QA/Constraints)과 관련 산출물(UCD, UCS, Glossary 등)을 SRS 문서 템플릿에 맞춰 조합·생성하는 기능.

| ID | Description | Status |
| ---- | ---- | ---- |
| FR-RQ-07-01 | 시스템은 확정된 요구사항과 산출물을 기반으로 SRS 문서를 생성할 수 있는 인터페이스("SRS 생성" 버튼 등)를 제공해야 한다. |  |
| FR-RQ-07-02 | 시스템은 SRS 문서를 표준 템플릿(IEEE 830 등) 구조에 맞춰 생성해야 한다. (Introduction, Overall Description, FR, NFR, Constraints, Use Cases, Glossary 등) |  |
| FR-RQ-07-08 | 시스템은 SRS 생성 시 요구사항 섹션(그룹) 구조가 존재하면, 섹션명을 하위 챕터 제목으로 반영해야 한다. (예: 섹션 "인증/로그인" → 3.1 인증/로그인) (FR-RQ-01-29 참조) |  |
| FR-RQ-07-03 | 시스템은 SRS 생성 시 프로젝트 개요(Overview)와 용어정의(Glossary)를 자동으로 포함해야 한다. |  |
| FR-RQ-07-04 | 시스템은 SRS 생성 시 UCD/UCS가 존재하는 경우 해당 내용을 SRS에 포함해야 한다. |  |
| FR-RQ-07-05 | 시스템은 SRS 생성 시 요구사항 간 누락이나 불일치가 있으면 사용자에게 경고해야 한다. |  |
| FR-RQ-07-06 | 시스템은 생성된 SRS 문서를 미리보기(Preview)할 수 있는 인터페이스를 제공해야 한다. |  |
| FR-RQ-07-07 | 시스템은 SRS 문서 생성 진행 상태를 실시간으로 표시해야 한다. |  |
