# Functional Requirements (FR-TC) — TC System

## FR-TC-01: TC 생성 공통 기능

| ID | Description | Status |
|---|---|---|
| FR-TC-01-01 | 시스템은 TC 생성 시 기본 기법(동치 분할, 경계값 분석)을 적용하여 TC를 생성해야 한다. | |
| FR-TC-01-02 | 시스템은 추가 테스트 설계 기법을 사용자에게 제시하고 선택받을 수 있어야 한다. | |
| FR-TC-01-03 | 시스템은 선택된 기법이 개별 요구사항에 적용 가능한지 판단하고, 적용 불가 시 사유를 안내해야 한다. | |
| FR-TC-01-04 | 시스템은 업로드할 Platform의 형식에 맞게 생성해야 한다. (Default JIRA) | |
| FR-TC-01-05 | 시스템은 사용자가 생성된 TC를 추가/수정/삭제할 수 있는 인터페이스를 제공해야 한다. | |
| FR-TC-01-06 | 시스템은 생성된 TC들을 선택할 수 있는 인터페이스를 제공해야 한다. | |
| FR-TC-01-07 | 시스템은 생성된 TC를 수정할 수 있는 'chat' 인터페이스를 제공해야 한다. (ex. "생성된 XX번 TC에 XXX 방식으로 3개 더 만들어줘" 같은 요구사항 처리) | |
| FR-TC-01-08 | 시스템은 생성된 TC에 대하여, 시스템 종료 후 재접속 후에도 생성된 TC를 유지하고 있어야 한다. ※ 저장을 하지 않아도 최후에 나온 TC는 유지한다 | |
| FR-TC-01-09 | 시스템은 생성된 TC가 어떤 요구사항으로부터 파생되었는지 매핑 관계를 사용자에게 표시해야 한다. | |

## FR-TC-02: 독립모드 고유기능

> ※ SAD를 생성하는 Flow와 관계없이 단독으로 TC를 만들고 싶은 유저. 가지고 있는 요구사항을 입력하면 TC를 만들어준다.

| ID | Description | Status |
|---|---|---|
| FR-TC-02-01 | 시스템은 공통 Import 모듈(FR-PF-05)을 활용하여 요구사항 문서를 업로드할 수 있어야 한다. | |
| FR-TC-02-02 | 시스템은 사용자가 자연어로 요구사항을 직접 입력할 수 있는 인터페이스를 제공해야 한다. | |
| FR-TC-02-03 | 시스템은 Import 또는 입력된 요구사항을 FR/QA/Constraints로 자동 분류한 후 TC를 생성해야 한다. | |
| FR-TC-02-04 | 시스템은 자동 분류된 FR/QA/Constraints 결과를 사용자에게 확인할 수 있는 인터페이스를 제공해야 한다. | |
| FR-TC-02-05 | 독립모드에서는 "TC 버전 관리"와 "요구사항 추적성" 기능을 제공하지 않는다. | |

## FR-TC-03: AISE2.0 요구사항 기반 TC 생성 (연동 모드)

> ※ AISE System을 이용하여 SRS를 만들고, 저장하여 그로부터 TC를 생성하는 모드

| ID | Description | Status |
|---|---|---|
| FR-TC-03-01 | 시스템은 AISE Project에 저장된 요구사항 목록을 조회 / 선택할 수 있어야 한다. | |
| FR-TC-03-02 | 시스템은 전체 요구사항 또는 선택된 요구사항에 대해 TC 생성을 요청할 수 있는 인터페이스를 제공해야 한다. | |

## FR-TC-04: Review 기능

| ID | Description | Status |
|---|---|---|
| FR-TC-04-01 | 시스템은 TC가 작성되기 이전 요구사항 및, 작성된 모든 TC에 대해 Review할 수 있는 기능을 가지고 있어야 한다. | |
| FR-TC-04-02 | 시스템은 사용자가 입력한 요구사항이 TC를 만들기 적절한지 판단할 수 있어야 한다. | |
| FR-TC-04-03 | 시스템은 TC간 충돌 / 중복 / 모호성을 검토해야 한다. | |
| FR-TC-04-04 | 시스템은 TC가 전체 요구사항을 커버하고 있는지 검토해야 한다. (목표율 80%) | |

## FR-TC-06: TC 버전 관리

> ※ 연동 모드에서만 제공 (독립모드에서는 FR-TC-02-05에 의해 제외)

| ID | Description | Status |
|---|---|---|
| FR-TC-06-01 | 시스템은 TC를 저장할 수 있는 기능을 제공해야 한다. | |
| FR-TC-06-02 | 시스템은 각 버전에 버전 번호, 생성 일시, 생성자를 기록해야 한다. | |
| FR-TC-06-03 | 시스템은 두 버전 간의 차이점을 비교할 수 있어야 한다. | |
| FR-TC-06-04 | 시스템은 변경 이력을 시간순으로 조회할 수 있어야 한다. | |
| FR-TC-06-05 | 시스템은 이전 버전으로 전체 복원할 수 있어야 한다. | |

## FR-TC-07: 요구사항 추적성

> ※ 연동 모드에서만 제공 (독립모드에서는 FR-TC-02-05에 의해 제외)

| ID | Description | Status |
|---|---|---|
| FR-TC-07-01 | 시스템은 TC와 현재 생성을 요청한(저장된, 버전정보가 있는) 요구사항 간의 연결을 생성할 수 있어야 한다. | |
| FR-TC-07-02 | 시스템은 'save' 버튼으로 새로 저장된 버전의 TC도 FR-TC-07-01의 요구사항을 만족해야 한다. | |
| FR-TC-07-03 | 시스템은 TC와 연결된 요구사항의 버전이 변경된 경우, 해당 TC가 최신 요구사항 기반이 아님을 사용자에게 표시해야 한다. | |
| FR-TC-07-04 | 시스템은 TC와 연결된 요구사항의 버전이 변경된 경우, 변경된 요구사항을 기반으로 TC를 재생성할 수 있는 인터페이스를 제공해야 한다. | |
