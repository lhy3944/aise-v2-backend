# UC-01: PRD Import를 통한 SRS 문서 생성

---

## 개요

| 항목 | 내용 |
|------|------|
| Use Case ID | UC-01 |
| 이름 | PRD Import를 통한 SRS 문서 생성 |
| Actor | 프로젝트 멤버 (Editor 이상) |
| 목적 | 기존 PRD 문서를 Import하여 요구사항을 추출하고, SRS 문서를 생성한다 |
| 사전조건 | 사용자가 로그인되어 있고, 프로젝트가 생성되어 있음 |
| 사후조건 | Review 완료된 SRS 문서가 저장되어 있음 |

---

## 기본 흐름 (Happy Path)

```
① 로그인/프로젝트 선택
       │
       ▼
② PRD 문서 업로드 (40p PDF)
       │
       ▼
③ 문서 파싱 + 미리보기
       │
       ▼
④ Classification 요청
       │
       ▼
  ┌─ 1차 pass: 문서 구조 파악 ─┐
  └─ 2차 pass: 섹션별 추출 ────┘
       │
       ▼
⑤ 분류 결과 검토 (FR/QA/Constraints/Other)
       │  ├─ 항목 수정/삭제
       │  ├─ 신규 항목 추가
       │  ├─ Other → 재분류
       │  └─ 낮은 신뢰도 항목 확인
       │
       ▼
⑥ 요구사항 확정 (저장)
       │
       ▼
⑦ "SRS 생성" 버튼 클릭
       │
       ▼
⑧ SRS 문서 생성 (템플릿 기반)
       │
       ▼
⑨ SRS Review (AI)
       │  ├─ 완전성/일관성/명확성 검토
       │  └─ 수정 제안 수락/거절
       │
       ▼
⑩ SRS 문서 확정 + 저장/Export
```

---

## 단계별 상세

### ① 로그인 / 프로젝트 선택

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SSO 로그인 → 대시보드 → 프로젝트 선택 (또는 신규 생성) |
| 시스템 응답 | 프로젝트 워크스페이스 진입, 모듈에 맞는 메뉴 표시 |
| 관련 요구사항 | FR-PF-01, FR-PF-02, FR-PF-09 |

### ② PRD 문서 업로드

| 항목 | 내용 |
|------|------|
| 사용자 행동 | Import 메뉴에서 40페이지 PDF 파일 업로드 |
| 시스템 응답 | 파일 수신, 업로드 완료 표시 |
| 관련 요구사항 | FR-PF-05-01 |
| 대안 흐름 | Jira/Polarion에서 직접 가져오기 (FR-PF-05-04, 05-05) |

### ③ 문서 파싱 + 미리보기

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 파싱 결과 미리보기 확인 |
| 시스템 응답 | PDF에서 텍스트/테이블/섹션 구조 추출, 미리보기 표시 |
| 관련 요구사항 | FR-PF-05-02, FR-PF-05-03 |
| 예외 흐름 | 파싱 실패 시 원인 표시 (FR-PF-05-06) |

### ④ Classification 요청 (2-pass)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | "Classification 시작" 버튼 클릭 |
| 시스템 응답 | **1차 pass**: 목차/섹션 구조 파악 → **2차 pass**: 섹션별 FR/QA/Constraints/Other 추출. 진행률 실시간 표시 |
| 관련 요구사항 | FR-RQ-06-01 ~ 06-12 |
| 예외 흐름 | 분류 실패 시 사유 표시 (FR-RQ-06-13) |

### ⑤ 분류 결과 검토

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 분류된 항목을 카테고리별로 검토. 수정/삭제/추가/재분류 |
| 시스템 응답 | 분류 결과를 요구사항 편집 화면에 반영. 신뢰도 낮은 항목 하이라이트. 중복 항목 병합 제안 |
| 관련 요구사항 | FR-RQ-06-05, 06-06, 06-09, 06-10, 06-11 |
| 비고 | AI Assist로 문장 정제 가능 (FR-RQ-01-02), 원본과 비교 가능 (FR-RQ-01-03) |

### ⑥ 요구사항 확정

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 최종 요구사항 선택 후 저장 |
| 시스템 응답 | 선택된 요구사항만 저장, 버전 생성 |
| 관련 요구사항 | FR-RQ-01-07, 01-08, FR-RQ-09 |

### ⑦ "SRS 생성" 버튼 클릭

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SRS 생성 버튼 클릭 |
| 시스템 응답 | 요구사항 누락/불일치 경고 (있을 경우). 생성 시작 |
| 관련 요구사항 | FR-RQ-07-01, 07-05 |
| 비고 | UCD/UCS가 이미 있으면 함께 포함 (FR-RQ-07-04) |

### ⑧ SRS 문서 생성

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 생성 진행 상태 확인, 완료 후 미리보기 |
| 시스템 응답 | IEEE 830 등 템플릿 기반 SRS 조합. 프로젝트 개요 + Glossary 자동 포함. 진행률 표시 |
| 관련 요구사항 | FR-RQ-07-02, 07-03, 07-06, 07-07 |

### ⑨ SRS Review (AI)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | Review 요청 → 결과 확인 → 수정 제안 수락/거절 |
| 시스템 응답 | 완전성, 일관성, 명확성 검토. 섹션별 수정 제안 제시 |
| 관련 요구사항 | FR-RQ-08-01 ~ 08-04, FR-RQ-02-07 |
| 대안 흐름 | 수정 반영 후 SRS 재생성 (FR-RQ-08-05) → ⑧로 돌아감 |

### ⑩ SRS 문서 확정 + Export

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SRS 저장 → 필요 시 PDF/Word/Markdown Export |
| 시스템 응답 | 버전 생성, Export 파일 다운로드 |
| 관련 요구사항 | FR-RQ-09, FR-PF-05-07, 05-08 |
| 대안 흐름 | Jira/Polarion으로 업로드 (FR-PF-05-09, 05-10) |

## 예외 흐름

| ID | 조건 | 처리 |
|----|------|------|
| EX-01 | 업로드 파일 형식 미지원 | 지원 형식 안내 후 재업로드 유도 |
| EX-02 | 문서 파싱 실패 (스캔 이미지 PDF 등) | 실패 원인 표시, 텍스트 기반 문서로 재업로드 안내 |
| EX-03 | Classification 중 일부 섹션 처리 실패 | 성공한 섹션 결과 표시 + 실패 섹션 재시도 옵션 |
| EX-04 | 요구사항 0건 추출 | 문서 내용이 요구사항 형태가 아님을 안내, 수동 입력 유도 |
| EX-05 | SRS 생성 시 필수 항목 부족 | 부족한 항목 목록 표시, 보완 후 재시도 안내 |
| EX-06 | SRS Review 후 대폭 수정 필요 | Review 결과 반영 → SRS 재생성 루프 (⑧↔⑨ 반복) |
