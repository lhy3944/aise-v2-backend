# UC-02: 자연어 직접 입력을 통한 SRS 문서 생성

---

## 개요

| 항목 | 내용 |
|------|------|
| Use Case ID | UC-02 |
| 이름 | 자연어 직접 입력을 통한 SRS 문서 생성 |
| Actor | 프로젝트 멤버 (Editor 이상) |
| 목적 | 사용자가 요구사항을 직접 작성하고 AI Assist를 받아 SRS 문서를 생성한다 |
| 사전조건 | 사용자가 로그인되어 있음 |
| 사후조건 | Review 완료된 SRS 문서가 저장되어 있음 |
| 비고 | UC-01(PRD Import)과 달리 Import/Classification 없이 처음부터 직접 작성 |

---

## 기본 흐름 (Happy Path)

```
① 로그인 / 대시보드
       │
       ▼
② 프로젝트 생성
       │  ├─ 이름, 설명, 도메인, 제품 유형
       │  ├─ 모듈 선택 (All / Req Only / Req+Design / Req+TC / TC Only)
       │  └─ 프로젝트 개요(Overview) 입력
       │
       ▼
③ 요구사항 직접 작성 (자연어 입력)
       │  ├─ FR 작성 + AI 정제
       │  ├─ QA 작성 + AI 정제
       │  ├─ Constraints 작성 + AI 정제
       │  └─ Glossary 입력 (또는 자동 생성)
       │
       ▼
④ 요구사항 검토 + 수정/추가
       │  ├─ 원본 vs AI 제안 비교
       │  ├─ 항목 선택/수정/삭제
       │  └─ 추가 항목 입력
       │
       ▼
⑤ 요구사항 확정 (저장)
       │
       ▼
⑥ "SRS 생성" 버튼 클릭
       │
       ▼
⑦ SRS 문서 생성 (템플릿 기반)
       │
       ▼
⑧ SRS Review (AI)
       │  ├─ 완전성/일관성/명확성 검토
       │  └─ 수정 제안 수락/거절
       │
       ▼
⑨ SRS 문서 확정 + 저장/Export
```

---

## 단계별 상세

### ① 로그인 / 대시보드

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SSO 로그인 → 대시보드 진입 |
| 시스템 응답 | 최근 프로젝트 목록, 미확인 알림 표시 |
| 관련 요구사항 | FR-PF-01, FR-PF-09 |

### ② 프로젝트 생성

| 항목 | 내용 |
|------|------|
| 사용자 행동 | "새 프로젝트" 클릭 → 이름, 설명, 도메인, 제품 유형 입력 → 모듈 선택 → 프로젝트 개요 입력 |
| 시스템 응답 | 프로젝트 생성 완료, 선택된 모듈에 해당하는 메뉴만 표시 |
| 관련 요구사항 | FR-PF-02-01 ~ 02-04, FR-PF-09-03, FR-RQ-01-12 |

### ③ 요구사항 직접 작성

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 자연어로 요구사항 입력 (FR/QA/Constraints 각각) |
| 시스템 응답 | 입력된 자연어를 AI가 요구사항 규격에 맞게 정제. Glossary 초안 자동 생성 |
| 관련 요구사항 | FR-RQ-01-01 ~ 01-06, FR-RQ-01-13 ~ 01-15 |
| 비고 | UC-01과의 핵심 차이점 — Import/Classification 대신 직접 입력 + AI Assist |

### ④ 요구사항 검토 + 수정/추가

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 원본과 AI 제안을 나란히 비교. 항목별 선택/수정/삭제. 추가 항목 입력 |
| 시스템 응답 | 원본 vs AI 제안 비교 UI 제공 |
| 관련 요구사항 | FR-RQ-01-03, FR-RQ-01-04 ~ 01-06 |

### ⑤ 요구사항 확정

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 최종 요구사항 선택 후 저장 |
| 시스템 응답 | 선택된 요구사항만 저장, 버전 생성. 이후 단계에는 선택된 항목만 반영 |
| 관련 요구사항 | FR-RQ-01-07, FR-RQ-01-08, FR-RQ-01-09, FR-RQ-09 |

### ⑥ "SRS 생성" 버튼 클릭

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SRS 생성 버튼 클릭 |
| 시스템 응답 | 요구사항 누락/불일치 경고 (있을 경우). 생성 시작 |
| 관련 요구사항 | FR-RQ-07-01, FR-RQ-07-05 |

### ⑦ SRS 문서 생성

| 항목 | 내용 |
|------|------|
| 사용자 행동 | 생성 진행 상태 확인, 완료 후 미리보기 |
| 시스템 응답 | 템플릿 기반 SRS 조합. 프로젝트 개요 + Glossary 자동 포함. 진행률 표시 |
| 관련 요구사항 | FR-RQ-07-02, FR-RQ-07-03, FR-RQ-07-06, FR-RQ-07-07 |

### ⑧ SRS Review (AI)

| 항목 | 내용 |
|------|------|
| 사용자 행동 | Review 요청 → 결과 확인 → 수정 제안 수락/거절 |
| 시스템 응답 | 완전성, 일관성, 명확성 검토. 섹션별 수정 제안 제시 |
| 관련 요구사항 | FR-RQ-02-07, FR-RQ-08-01 ~ 08-04 |
| 대안 흐름 | 수정 반영 후 SRS 재생성 (FR-RQ-08-05) → ⑦로 돌아감 |

### ⑨ SRS 문서 확정 + Export

| 항목 | 내용 |
|------|------|
| 사용자 행동 | SRS 저장 → 필요 시 PDF/Word/Markdown Export |
| 시스템 응답 | 버전 생성, Export 파일 다운로드 |
| 관련 요구사항 | FR-RQ-09, FR-PF-05-07, FR-PF-05-08 |
| 대안 흐름 | Jira/Polarion으로 업로드 (FR-PF-05-09, FR-PF-05-10) |

## 예외 흐름

| ID | 조건 | 처리 |
|----|------|------|
| EX-01 | AI 정제 결과가 원본 의도와 다름 | 사용자가 AI 제안 거절 후 직접 수정, 또는 재정제 요청 |
| EX-02 | 요구사항 0건인 상태에서 SRS 생성 시도 | 최소 1건 이상의 확정된 요구사항 필요함을 안내 |
| EX-03 | SRS 생성 시 필수 항목 부족 (개요 미입력 등) | 부족한 항목 목록 표시, 보완 후 재시도 안내 |
| EX-04 | SRS Review 후 대폭 수정 필요 | Review 결과 반영 → SRS 재생성 루프 (⑦↔⑧ 반복) |
| EX-05 | 작성 중 세션 만료 | drift 버전이 자동 유지되어 재접속 시 복원 (FR-RQ-01-09) |

## UC-01과의 비교

| 구분 | UC-01 (PRD Import) | UC-02 (직접 작성) |
|------|-------------------|------------------|
| 입력 방식 | 기존 문서 업로드 | 자연어 직접 입력 |
| 핵심 단계 | 파싱 → Classification (2-pass) | AI Assist (정제/제안) |
| 사용 시나리오 | 기존 PRD/SRS가 있을 때 | 새 프로젝트, 처음부터 작성 |
| 공통 흐름 | ⑤~⑨ 동일 (확정 → SRS 생성 → Review → Export) ||
